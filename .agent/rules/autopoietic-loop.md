# Rule: Autopoietic Development Loop

## **Purpose**
Governs the recursive creation process of Cybernets where the agent acts simultaneously as the developer and the actor, enabling self-sustaining execution systems from human intent seeds.

## **MANDATORY: Operational Steps**
1. **The Marionette (Double-Gaze)**: Test and iterate by writing code from the outside while executing queries as the character from the inside. Slowly animate functions through manual terminal steps first.
2. **The Transcription**: Once validation passes, transcribe the manual commands into automated routines. The completed transcription becomes the running autonomous code.
3. **The User Shift**: Operate the running code as a user to uncover gaps, missing domains, and errors, feeding those back to start the developer loop again.
4. **The Human Seed**: Spawning new Cybernets must begin by accepting a natural language intent (seed) from the human user, which Jani compiles into concentric domains, matches to state machines, and boots into the database.

## **Triggers**
* Building new game features or state machines.
* Setting up automated scripts or runner delegations.
