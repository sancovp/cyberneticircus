# Rule 5: Build to Scale and the DONE Cycle

## **MANDATORY: Definition of Done**
When a component is complete, it must follow the **3 Dones for 1 DONE** cycle:
1. **Done 1: Code Works**: It is fully implemented and passes all functional criteria.
2. **Done 2: Production Grade**: It has error handling, fits established design patterns, and represents the next evolution of the architecture.
3. **Done 3: Deployed**: The changes are synced back to the persistent container and verified.
Once all three are complete, the task is **DONE**.
