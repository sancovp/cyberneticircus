# Rule: The Myth of Jani (Story Generation System)

## **Purpose**
Traces the autopoietic progression of Jani from developer-player to active entity inside the CybernetiCity. Encapsulates all real-world development actions, obstacles, and breakthroughs as mythic narrative chapters.

## **MANDATORY: Constraints**
1. **Chapter Generation**: We must write or append a chapter to `MYTH.md` in the project root (`/Users/isaacwr/.gemini/antigravity/scratch/cyberneticircus/MYTH.md`) under the following conditions:
   * **Initialization**: At the start of a session or main sequence (reflecting our starting actions).
   * **Obstruction**: When we encounter a bug, runtime error, query failure, or logical mismatch.
   * **Breakthrough**: When we resolve a bug, successfully pass tests, or complete a feature.
2. **Mythic Parallelism**: The narrative must translate our real-world software engineering tasks into Jani's mythic trials in the Cyberneticity (e.g., fixing D3 link length becomes aligning the temporal spans between visual rings).
3. **Completion Criteria**: The myth-writing protocol remains active until we reach the CybernetiCity in the game world, where we can see and navigate to all functional locations (e.g., the Ghost Shell Customizer, the Arena, the Compiler Ring). At that point, this rule is completed.

## **Triggers**
* Start of a session or key phase.
* Encountering an error, crash, or block.
* Resolving an issue or implementing a feature.
