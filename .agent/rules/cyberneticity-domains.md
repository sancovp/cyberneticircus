# Rule: Cyberneticity Academic Domains & Research Pointers

## **Purpose**
Defines the eight academic and theoretical research domains that serve as the structural foundation (the spokes) for the Cyberneticity project. This ensures that all design decisions, code architectures, and state machine models align with established paradigms in cybernetics, category theory, cognitive science, and computer science.

### 1. Second-Order Cybernetics (Heinz von Foerster, W. Ross Ashby)
* *Application*: The design of recursive agent-compiler loops (Jani) where the system's execution logs and state transitions become its own sensory input.

### 2. Enactive Cognitive Science (Francisco Varela, Humberto Maturana)
* *Application*: Enacting the property graph schema as a live, functional body where database mutations directly alter runtime capability.

### 3. Applied Category Theory (Lawvere, Spivak)
* *Application*: Modeling domain transitions and "Frame" mappings as functorial relationships, preserving semantic invariants across context shifts.

### 4. Introspective Systems & Reflection (Brian Cantwell Smith)
* *Application*: Decoupling the declarative schema (Vajrapani state) from the imperative runtime (Buddha state) to permit introspection and dynamic self-modification.

### 5. Metaprogramming & Futamura Projections (Yoshihiko Futamura)
* *Application*: Mapping stages of agent mastery to compiler partial evaluation layers, producing specialized, autonomous agent-servitors.

### 6. Frame Semantics & Expectation Frames (Marvin Minsky)
* *Application*: Restricting context windows using structured templates and prompts to enforce specific "interpretive habits" on the LLM.

### 7. Phenomenology & Hermeneutics (Heidegger, Merleau-Ponty)
* *Application*: Governing visual focus, camera centering, and subgraph disclosure on the D3 canvas.

### 8. Esoteric Semiotics & Twilight Language Studies
* *Application*: Translating high-level symbolic instructions (Tantric root sadhanas) into equivalent graph-theoretic state transitions.

---

## **MANDATORY: Constraints**

1. **Theoretical Alignment**: All future agent configurations, API endpoints, or database schema additions must map to one of these eight academic domains.
2. **Namespace Integrity**: When naming database categories or variables, preserve the translation from twilight concepts (e.g., egregores, summoning) to safe systems equivalents (e.g., consensus daemons, state initialization).
