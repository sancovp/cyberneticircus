FAQ

Frequently asked questions.

How does CybernetiCircus prevent agents from getting stuck in loops? ↓

We wrap agent actions in structured state machine traversal steps. If an agent repeats queries or exceeds pre-defined turn boundaries, the call stack is automatically halted or redirected to a fallback state, preventing runaway token costs.

Is CybernetiCircus model-agnostic? ↓

Yes. It operates as an execution middleware layer. You can use any foundation LLM (Claude, Gemini, OpenAI, or local models) to generate decisions, while CybernetiCircus validates and tracks state transitions in the graph database.

What database does the platform require? ↓

It runs natively on Neo4j to store agent identities, state machines, and wiki pages as a unified graph. This allows the visualizer to map relationships and page satellite clusters dynamically.

Is it difficult to self-host? ↓

No. The entire system is packaged in a single-command docker-compose stack. Running docker-compose up -d --build boots the web server, D3 visualizer canvas, and Neo4j database instances instantly.

How does the visualizer handle very large agent memory graphs? ↓

It uses specialized D3 clustering forces to organize wiki pages, document blocks, and agent state nodes into visual districts, preventing the layout from becoming a cluttered hairball and keeping retrieval fast.
